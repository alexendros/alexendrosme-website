## 0. Resumen ejecutivo

El sitio es desktop-first con `md:768px` como único breakpoint real. Tres problemas **P0 (bloquean test de 5 s / rompen UX)** y varios **P1/P2** detectados. Seis acciones de código concentran el 90 % del fix. Performance y a11y base son buenas (CLS=0, cero errores JS, cero imágenes) — el problema es de layout, no de rendimiento.

**Veredicto**: antes de tocar código hay 3 problemas P0 que deberían entrar en un único PR móvil. Los P1/P2 pueden ir en un segundo PR.

## 4. Verificación post-implementación

Después de aplicar cada PR, repetir con Chrome DevTools MCP:

1. **VP1–VP3**: `document.documentElement.scrollWidth === window.innerWidth` en las 8 URLs. Debe ser `true`.
2. **Test de 5 s**: en `/` con VP1, H1 + subcopy + al menos un CTA visibles sin scroll.
3. **Tap targets**: `Array.from(document.querySelectorAll('a,button')).filter(e => { const r=e.getBoundingClientRect(); return (r.width<44||r.height<44) && r.width>0; }).length === 0` en todas las URLs con VP1.
4. **Lighthouse mobile** en `/`: score ≥ 90 en Performance y A11y.
5. **Tabla /about**: con VP1, `table.scrollWidth <= wrapper.clientWidth` tras aplicar reflow a lista, o bien la sombra lateral es visible.
6. **Sheet drawer VP1**: `sheet.getBoundingClientRect().width / window.innerWidth <= 0.8`.
7. **Regresión visual**: comparar screenshots pre/post en las 5 × 8 combinaciones.

## 6. Riesgos de la implementación

- **`px-4 sm:px-6` global**: cambio amplio. Asegurar que los `<Separator mx-auto max-w-3xl px-6/>` se actualicen también para que la línea visual coincida con el contenido.
- **`clamp()` en H1**: revisar que el `[text-shadow]` y el `tracking-tight` sigan funcionando con el tamaño menor. El efecto de halo verde puede saturar a 28 px.
- **`h-11` en Button default**: no tocar el `size: default` base — añadir `size: touch` aparte. Cambiar el default rompería el layout de los 15+ botones del sitio.
- **Sheet width**: al reducir a `w-[min(18rem,85vw)]`, validar que el contenido (logo + 5 links) no se desborde a 85vw en VP1 (85% × 320 = 272 px).
- **Tabla /about → lista**: si se decide reflow total bajo `sm:`, preservar la jerarquía semántica (no usar `<table>` truncada; usar `<dl>` o `<ul>` explícitos por fila para SEO y a11y).
